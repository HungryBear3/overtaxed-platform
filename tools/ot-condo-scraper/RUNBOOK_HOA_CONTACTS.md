# OT HOA / Condo-Board Contact Pipeline — Runbook

This is the simple side of the OT outreach prospect work — a deterministic
exporter that turns a hand-curated seed JSON into a reviewable CSV. It exists
because the live `scrape.ts` Realie path is currently blocked (missing
`outreach_prospects` table, stale `api.realie.io` URL, Prisma v7 init issue).
The exporter sidesteps all three so we can produce a reviewable HOA prospect
list today.

## What this is and isn't

| | This exporter | The existing `scrape.ts` |
|---|---|---|
| DB writes | None | Upserts `outreach_prospects` |
| External HTTP | None | Calls Realie |
| Secrets needed | None | `DATABASE_URL`, `OT_REALIE_API_KEY` |
| Input | `data/hoa-prospects-seed.json` | Realie API |
| Output | CSV at workspace docs path | Prisma rows |
| Sends emails? | **No.** | No — but feeds the sender pipeline downstream |

## Refresh / day-2 workflow

When new public-source evidence surfaces for an existing prospect, or when a
new management company / association needs to be tracked:

1. **Open the seed:** `tools/ot-condo-scraper/data/hoa-prospects-seed.json`.
2. **Edit or append** rows. Every row must include:
   - `organization_name` (non-empty)
   - `category` — one of `hoa_management_company`, `condo_association`,
     `board_member_lead`, `directory_resource`
   - `confidence` — one of `high`, `medium`, `low`
   - `source_url` — at least one public URL (space-separate multiples)
   - `outreach_status` — defaults to `needs_review`; only promote to `ok`
     after Alexy reviews the contact path
3. **Dry-run first** to see what would land in the CSV:
   ```bash
   pnpm tsx tools/ot-condo-scraper/export-hoa-contacts.ts --dry-run
   ```
4. **Write the CSV** (overwrites the file at the workspace docs path):
   ```bash
   pnpm tsx tools/ot-condo-scraper/export-hoa-contacts.ts
   ```
5. **Hand-review** the CSV before any downstream use. Spot-check 3–5
   `source_url` values to confirm they still resolve.

## Confidence rubric

| Tier | Meaning | Example |
|---|---|---|
| `high` | Named public business role + public business contact (email/phone), corroborated by 2+ public sources. | Hillcrest's Amy Haith, President — listed on company team page + LinkedIn + public `info@` inbox. |
| `medium` | Org identified, public business phone OR contact form, no specific named role yet. | Foster Premier — strong public phone, contact form only. |
| `low` | Org named via single directory listing, contact path not yet verified. | Anything where you have just one source URL and no contact surface. |

## Outreach-status rubric

| Status | Meaning |
|---|---|
| `needs_research` | Org named, not yet investigated. |
| `needs_review` | Public source(s) found, but Alexy must approve channel + recipient before any outreach. **Default for every row.** |
| `ok` | Cleared by Alexy for inclusion in a pilot send. The exporter refuses to mark a row `ok` unless `source_url` is non-empty. |

## Hard rules (carried from the prompt)

- **No outreach** runs from this script. CSV is the deliverable; sending is a
  separate, gated decision.
- **No CRM writes** unless explicitly approved. This script does not touch
  Prisma, Supabase, or any database.
- **No ToS-risky scraping.** Use only public directory pages, company "about"
  pages, CAI-Illinois directory, and similar. Do not scrape Yelp behind auth
  walls; do not bypass robots.txt; do not buy lists from data brokers.
- **No secrets in repo.** This script reads no env vars beyond `--seed` /
  `--out` / `--dry-run`.
- **Do not use parcel owner names as contacts.** Owner-of-record on a Cook
  County PIN is private homeowner data and is not an outreach target.

## When to switch back to the Realie-based scraper

Three blockers must clear before `scrape.ts` is usable again:

1. **Prisma model `outreachProspect` + DB table `outreach_prospects`** exist
   and `npx prisma migrate status` is clean.
2. **`OT_REALIE_BASE_URL`** is set to a currently-working Realie endpoint
   (the default `api.realie.io` is stale; OT app code already uses
   `https://app.realie.ai/api/public/...` patterns).
3. **Prisma client init** under v7 no longer throws on instantiation; this
   is currently blocked by client options drift.

Until then, the seed-JSON path is the canonical lane.

## Adding a new target — minimal checklist

1. Find a non-Yelp public source: the company's own `/about`, `/team`,
   `/contact` page, or the CAI-Illinois member directory.
2. Capture the source URL into `source_url`.
3. Use the company's published business phone / public business inbox if
   they exist on that public page. Never invent emails. Never use parcel
   owner names.
4. Set `confidence` per the rubric above; `outreach_status` stays
   `needs_review` until Alexy decides.
5. Run `--dry-run`, eyeball the row, then write the CSV.
